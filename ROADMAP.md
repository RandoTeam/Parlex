# Parlex Roadmap

Рабочее правило: не больше одной фазы за раз. Каждая фаза заканчивается
коротким отчетом, проверкой на устройстве там, где это важно, и отдельным
коммитом перед переходом к следующей фазе.

## Камера

### Фаза 12: Полноразмерный снимок

- Заменить capture через `PreviewView.bitmap` на CameraX `ImageCapture`.
- OCR снимка выполнять по полноразмерному кадру, а не по preview surface.
- Live OCR оставить на `ImageAnalysis`; capture OCR вынести на `ImageCapture`.
- Проверить на двух подключенных телефонах с русским и английским печатным текстом.

Ожидаемый результат: более резкий вход для OCR, лучшее распознавание кириллицы,
меньше пропущенных строк.

### Фаза 13: Управление камерами и светом

- Добавить компактную панель управления камерой в live-режиме.
- Добавить включение/выключение фонаря через `CameraControl.enableTorch`.
- Добавить режим вспышки для `ImageCapture`, если устройство его поддерживает.
- Добавить переключение доступных камер:
  - основная задняя камера по умолчанию;
  - фронтальная камера, если она доступна;
  - ультраширик / телевик, если телефон отдает их как отдельные camera ID;
  - аккуратный fallback, если телефон показывает только одну логическую заднюю камеру.
- Сохранять выбранную камеру только если она доступна на текущем устройстве.

Ожидаемый результат: камера лучше работает при плохом освещении и может использовать
подходящий объектив для меню, вывесок, документов и близкого текста.

### Фаза 14: Debug capture pack

- Добавить debug-действие для сохранения:
  - оригинального кадра;
  - OCR backend и языка;
  - OCR boxes по строкам;
  - распознанного исходного текста;
  - перевода;
  - итоговой картинки с наложением.
- Держать это локальным инструментом и не показывать в обычном пользовательском flow.

Ожидаемый результат: проблемы камеры чинятся по реальным примерам с телефона,
а не по догадкам.

### Фаза 15: Умный OCR fallback

- Определять скрипт по результатам OCR и дополнительным проходам.
- Пробовать Cyrillic/Latin fallback, если выбранный source language не совпадает с кадром.
- Предпочитать строки с подходящим скриптом и нормальной уверенностью.
- Сохранять стабильные координаты между fallback-проходами.

Ожидаемый результат: русский текст находится даже при неверно выбранном исходном языке,
а английский не уходит в Tesseract без необходимости.

### Фаза 16: Перевод блоками с сохранением строк

- Координаты наложения оставить построчными.
- Переводить крупные текстовые блоки для лучшего контекста.
- Возвращать перевод обратно по строкам через стабильные ID.
- Если модель не сохраняет структуру, откатываться на построчный перевод.

Ожидаемый результат: качество перевода выше, но размещение остается в стиле
Google/Yandex camera translate.

### Фаза 17: Полировка overlay

- Улучшить замазывание оригинального текста цветом окружения.
- Сделать предсказуемый минимум размера текста.
- Добавить перенос строк для высоких OCR-блоков.
- Улучшить контраст на светлом, темном и фактурном фоне.
- Держать UI чистым, без декоративного шума.

Ожидаемый результат: результат выглядит как продуктовый camera translate, а не как debug rectangles.

### Фаза 18: Tap line mode

- По нажатию на строку или блок открывать компактную панель.
- Показывать оригинал, перевод, копирование и озвучку.
- Оставлять камеру видимой за панелью.

Ожидаемый результат: камера полезна даже тогда, когда inline overlay вынужден сокращать
длинный перевод.

### Фаза 19: Quality meter

- Показывать короткие подсказки качества:
  - слабый фокус;
  - слишком мелкий текст;
  - мало света;
  - OCR не нашел стабильный скрипт;
  - модель перевода недоступна.
- Подсказки должны быть короткими и практичными.

Ожидаемый результат: пользователь понимает, где проблема: OCR, фокус, свет,
выбранный язык или качество модели.

## Модели и runtime

### Фаза 20: TranslateGemma LiteRT beta spike

- Оставить HY-MT и TranslateGemma GGUF как стабильные модели.
- Добавить TranslateGemma LiteRT как отдельный beta runtime.
- Сравнить на реальных телефонах:
  - размер модели;
  - время загрузки;
  - latency первого токена;
  - tokens per second;
  - RAM;
  - нагрев;
  - качество перевода.
- Держать beta скрытой или явно подписанной, пока она не даст реальный выигрыш над GGUF.

Ожидаемый результат: решение по LiteRT принимается по измерениям, а не по теории
про GPU/NPU.

### Фаза 21: Offline language packs

- Группировать переводческую модель, OCR assets, speech recognition и TTS readiness
  по практическим сценариям.
- Начать с пакетов:
  - Russian-English travel pack;
  - English-Russian camera pack;
  - Vietnamese-English travel pack.

Ожидаемый результат: пользователь понимает, что установлено и что полностью работает offline.

## Продуктовые режимы

### Фаза 22: Document mode

- Снимать страницу, меню или инструкцию.
- По возможности выравнивать перспективу.
- Делать OCR блоками.
- Показывать читаемый переведенный документ.

### Фаза 23: Travel / street mode

- Оптимизировать camera UX под вывески, меню, билеты, транспорт и короткий публичный текст.
- Добавить быстрые действия: сохранить, копировать, озвучить.

### Фаза 24: Live subtitle camera mode

- Держать стабильные переведенные субтитры для экранов, презентаций, вывесок и движущейся камеры.
- Использовать уже сделанные live text tracks и translation cache.

### Фаза 25: Auto source camera translate

- Добавить режим `Auto` для исходного языка камеры.
- Определять скрипт и вероятный язык по строкам/блокам OCR, а не только по одному выбранному языку.
- Выбирать OCR backend и translation source автоматически, но оставлять ручной override.
- Не блокировать перевод при слабой уверенности: камера должна переводить лучшую найденную гипотезу и только показывать подсказку.
- Сохранять в debug pack выбранный язык, альтернативные гипотезы и причину выбора.
- Первая версия работает по одному исходному языку на кадр; смешанные таблички остаются фазой 26.

Ожидаемый результат: пользователь может навести камеру на незнакомую вывеску без ручного выбора source language, а приложение все равно старается перевести кадр.

### Фаза 26: Mixed-language signs and tables

- Разбивать кадр на независимые OCR-группы, если на фото несколько языков: например English, Français, Deutsch, Čeština.
- Определять язык/скрипт для каждой строки или таблички отдельно.
- Переводить каждую группу своим source language в общий target language.
- Сохранять исходную геометрию строк, чтобы overlay ложился на конкретную табличку, а не смешивал весь текст в один контекст.
- Для таблиц отдельно сохранить rows/columns и не ломать порядок ячеек при переводе.
- В debug pack сохранять row/column для каждой строки, чтобы проверять порядок ячеек на реальных фото.
- Первая реализация работает в capture mode: live mode остается одноязычным для стабильной производительности.

Ожидаемый результат: камера сможет работать с меню, указателями и туристическими табличками, где рядом находятся несколько языков.

### Фаза 27: Gallery photo translate

- Добавить выбор фотографии из системной галереи без отдельного permission flow.
- Прогонять выбранное фото через тот же OCR, mixed-language detection, перевод и overlay, что и снимок камеры.
- Учитывать EXIF-поворот и уменьшать очень большие изображения до рабочего размера перед OCR.

Ожидаемый результат: пользователь может перевести уже сделанное фото, скриншот, меню или табличку без повторного наведения камеры.
